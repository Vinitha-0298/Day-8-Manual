# Day-8-Manual
Experience Based Testing Technique:
-----------------------------------
Experience-based testing techniques use the knowledge, intuition, and past experience of testers, developers, and stakeholders to identify defects in software.
These techniques are useful when:
      * Formal specifications are limited or absent
      * Time constraints require a faster testing approach
     
Types of Experience-Based Testing:
Error Guessing:
-----------------
This technique depends on the tester’s intuition and experience to predict where defects may occur.
Testers use:
      * Knowledge of common programming errors
      * Past system failures
      * Typical user behavior
to identify possible problem areas.
  
Exploratory Testing:
--------------------
In exploratory testing, the tester:
      * Learns the software
      * Designs test cases
      * Executes tests
all at the same time.
This is an unscripted approach that allows testers to:
      * Explore freely
      * Follow instincts
      * Focus on areas likely to contain defects

Checklist-Based Testing:
------------------------
 This technique uses predefined checklists to guide testing.
 The checklist is usually created from:
     * Previous projects
     * Industry best practices
 It reminds testers about:
     * Common test scenarios
     * Vulnerabilities
     * Important areas to test

Attack Testing:
---------------
Attack testing is also called:
     * Negative testing
     * Security testing
In this method, testers intentionally try to break the system by:
     * Giving invalid inputs
     * Exploiting vulnerabilities
     * Attempting unauthorized actions
This technique depends on the tester’s knowledge of:
    * Security principles
    * Common attack methods

Characteristics of Experience-Based Testing
Reliance on Human Expertise
---------------------------
These techniques depend on:
     * Tester skills
     * Knowledge
     * Experience

Flexibility and Adaptability
----------------------------
   * Less structured than formal testing
   * Allows changes and exploration during testing

Effective in Specific Situations
--------------------------------
Useful when:
     * Documentation is incomplete
     * Deadlines are tight
     * Systems are low risk

Coverage May Vary
-----------------
The effectiveness and coverage depend on:
     * Tester skill level
     * Tester experience

Scenarios When Experience-Based Testing Should Be Avoided
Contractual Requirements
-------------------------
Not suitable when projects require:
     * Clear test coverage
     * Specific test matrices
     * Strict documentation
Because experience-based testing cannot easily measure coverage.

Regulatory Compliance
---------------------
Not ideal for industries requiring:
    * Strict standards
    * Documented testing processes
Structured testing is more reliable in these cases.

Critical Systems
----------------
 Not recommended as the only testing method for systems where failure can cause serious consequences.
 Structured methods provide better assurance.

Complex Scenarios
-----------------
Experience-based testing may miss issues in:
    * Complex systems
    * Large scenarios
Structured testing provides better organization.

New Technologies
-----------------
For new technologies:
   * Limited past experience exists
   * Testing becomes difficult
Structured approaches help achieve better coverage.

Advantages of Experience-Based Testing:
----------------------------------------
 Works Well with Limited Documentation : Useful when detailed documents are unavailable.
 Effective Under Time Constraints      : Helps complete testing quickly within tight schedules.
 Provides Early Developer Feedback     : Helps developers fix issues faster.
 Improves Product Familiarity          : Testing teams understand the software better as it evolves.
 Useful for Operational Failures       : Effective for analyzing and fixing operational problems.
 Supports Multiple Testing Techniques  : Allows different testing methods based on project needs.
 Quick Exploratory Testing Start       : Does not require extensive predefined test plans.
 Fills Gaps in Automated Testing       : Tests areas that are difficult to automate.
 Uses Domain Expertise                 : Uses knowledge from:
                                              * Business analysts
                                              * Customers
                                              * Clients
                                              * Technical experts to improve testing quality.

Disadvantages of Experience-Based Testing
-----------------------------------------
  Not Suitable for Detailed Systems  : Not ideal for systems needing detailed documentation.
  Repeatability Issues               : Results may differ between testers.
  Difficult to Measure Coverage      : Test coverage is harder to calculate accurately.
  Less Suitable for Automation       : These tests are difficult to automate later.
  Depends on Tester Skill            : Testing quality depends heavily on tester expertise.

Flow of Experience-Based Testing:
Step 1: Test Session Preparation
--------------------------------
    * Testing begins with preparation.
    * This preparation:
    * Sets the stage for testing
    * May or may not be documented

Step 2: Immersive Test Sessions
-------------------------------
Testers work together in collaborative sessions.
   They:
     * Explore the system
     * Continuously test and learn

Step 3: Iterative Test Design
------------------------------
Testers design test cases during the session using:
      * Experience
      * Knowledge
      * Insights
The goal is to identify possible defects.

Step 4: Executing and Evaluating Tests
--------------------------------------
Testers:
    * Execute test cases
    * Observe system responses
    * Analyze outcomes
This step involves continuous probing and evaluation.

Step 5: Learning and Adapting
------------------------------
Testers learn from:
    * Test execution
    * System behavior
They improve and adjust testing strategies accordingly.

Step 6: Continuous Design and Iteration
----------------------------------------
Insights from testing help create:
   * New test scenarios
   * Better testing approaches
   * Testing continues in cycles.

Step 7: Debriefing and Reporting
--------------------------------
At the end of the session, testers share:
   * Observations
   * Findings
   * Defects
This improves communication with stakeholders.
